Introduction

What is 'IndexedDB'?
---
IndexedDB의 정의가 중요한게 아니니 간단하게만 짚고 넘어가자면, Web에서 사용하는 (key, value) 형식의 Database 구조이며 ```commit```과 ```rollback```이 가능한 ```Transaction Database```이다.  
  
CVE-2021-30633 Cause
---
~~~c++
bool IsAcceptingRequests() {
    return !is_commit_pending_ && state_ != COMMITTING && state_ != FINISHED;
}
~~~
위 코드는 사용자가 Transaction을 요청할 때 현재 Database 상태가 ```commit 보류 중``` 이거나 ```committing``` 이거나 ```commit finished``` 상태면 요청을 거부하는 메소드이다. 사실 ```Transaction Database```에서는 이러한 메소드가 있는 것이 당연하다. 만약 저러한 상태에서 추가적인 ```Transaction```을 수행하면 취약점이 발생할 수 있다. 

아래는 해당 코드가 추가된 ```TransactionImpl::Put``` 메소드이다.

우리는 이 버그를 트리거하기 위해 세 가지 상태 중 적어도 하나는 컨트롤 해야한다. ```COMMITTING```과 ```FINISHED``` 상태는 특정한 메소드를 호출하여 상태를 변경할 수 있는데, 아쉽게도 ```FINISHED``` 상태를 조작하기 위한 메소드에서 중간에 ```IndexedDBConnection::RemoveTransaction()```를 만나 트랜젝션이 제거되므로 ```COMMITTING``` 상태를 노려야 한다.  
  
```COMIITING``` 상태와 관련있는 메소드는 ```IndexedDBTransaction::Commit()``` 와 ```TransactionImpl::Commit()```가 있는데 ```IndexedDBTransaction::Commit()```에서 흥미로운 Function Call Chain이 존재한다. 

~~~c++
Function Call Chain: IndexedDBTransaction::Commit --> IndexedDBBackingStore::Transaction::CommitPhaseOne --> IndexedDBBackingStore::Transaction::WriteNewBlobs
~~~

여기서 ```Clone```을 이용해 js를 재진입하고 특정 작업을 트랜젝션에 push 할 것이다. ```bindInterface()```를 이용하여 ```Receiver```와 바인딩하여 ```Remote```를 통해 ```Browser Code```에 접근해야 한다.  

먼저 Javascript에서 ```fileAccessImpl```을 구현하고 ```__proto__```를 이용해 ```Clone()```을 재정의한다. 그 다음 ```mojo.Binding.bind```를 이용해 ```mojo.makeRequest()```에서 반환된 ```InterfaceRequest```를 우리가 구현한 ```fileAccessImpl```에 바인딩한다. 이렇게 하면 ```Render```에서 ```Browser```에게 ```PendingReceiver```을 보내지 않아도 되며, ```Browser```상에 코드를 ```Render```에서 호출할 수 있게 된다.  

이제 ```external object```를 선언하고 fileSystemAccessToken을 생성한 fileAccessPtr 객체와 연결한다. 
~~~c++
var external_object = new blink.mojom.IDBExternalObject();
external_object.fileSystemAccessToken = fileAccessPtr;
~~~
```IndexedDBBackingStore::Transaction::WriteNewBlobs()```에서 실행되는 메소드인 ```entry.file_system_access_token_remote()```를 통해 ```Remote```를 가져오고 이후 ```Clone()```은 우리가 재정의한 메소드로 동작한다.
~~~c++
entry.file_system_access_token_remote()->Clone(token_clone.InitWithNewPipeAndPassReceiver());
~~~
